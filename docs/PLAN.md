# ガントチャート ジェネレーター — 要件定義・設計書

## 概要

タスク・工数・担当者を入力として、リソース制約付きスケジューリングを行い、
ガントチャートを HTML / PNG でエクスポートできる Web アプリ。

バックエンド不要の単一 HTML ファイルとして動作する。

---

## 機能要件

### FR-1: プロジェクト基本設定

| ID | 要件 |
|----|------|
| FR-1-1 | プロジェクト名・開始日・リリース日を設定できる |
| FR-1-2 | リリース評価期間を **日数 / 週数 / 月数** のいずれかで指定できる |
| FR-1-3 | 休日として **祝日（国指定）** と **会社指定の休業日** を個別に登録・管理できる |

**FR-1-2 補足：**

```jsonc
"evalPeriod": { "value": 4, "unit": "weeks" }
// unit: "days" | "weeks" | "months"
// 月数の場合は暦月で計算し、営業日に換算する
```

**FR-1-3 補足：**

```jsonc
"holidays": {
  "national": ["2026-05-03", "2026-05-04"],   // 祝日（国指定）
  "company":  ["2026-12-29", "2026-12-30"]    // 会社指定休業日（年末年始・創立記念日など）
}
```

---

### FR-2: タスク管理

| ID | 要件 |
|----|------|
| FR-2-1 | アイテム（開発項目）を追加・編集・削除できる |
| FR-2-2 | アイテムにカテゴリ・メモを付与できる |
| FR-2-3 | アイテム内に複数フェーズを順番に定義できる（パイプライン） |
| FR-2-4 | フェーズに工数（営業日数）を設定できる |
| FR-2-5 | フェーズに「バックグラウンド（工数消費なし）」フラグを付けられる |
| FR-2-6 | フェーズに固定開始日を設定できる |

---

### FR-3: 担当者管理

| ID | 要件 |
|----|------|
| FR-3-1 | 担当者を追加・編集・削除できる |
| FR-3-2 | 担当者にチーム・担当可能フェーズタイプ・参画開始日・稼働率を設定できる |
| FR-3-3 | フェーズに「許可担当者（allowedPeople）」を指定できる |
| FR-3-4 | `requireAll` フラグで複数担当者の同時着手を要求できる |
| FR-3-5 | フェーズタイプ未指定の場合、担当可能な人を自動割り付ける |

---

### FR-4: 自動スケジューリング

| ID | 要件 |
|----|------|
| FR-4-1 | LPT（残パイプライン日数優先）アルゴリズムで優先度を自動計算する |
| FR-4-2 | 担当者の空き状況・参画開始日を考慮してタスクを配置する |
| FR-4-3 | 先行フェーズ完了後に後続フェーズを開始する（依存関係制御） |
| FR-4-4 | リリース評価開始日を超えるタスクを警告表示する |
| FR-4-5 | 祝日・土日・会社指定休業日を除いた営業日カレンダーでスケジューリングする |

---

### FR-5: 可視化

| ID | 要件 |
|----|------|
| FR-5-1 | ガントチャート（SVG）を描画する |
| FR-5-2 | フェーズタイプごとの色設定ができる |
| FR-5-3 | スケジュールサマリー一覧表（担当者・開始日・終了日・日数）を表示する |
| FR-5-4 | 凡例を表示する |
| FR-5-5 | 警告（リリース評価超過・担当者未割り当て等）をバナー表示する |
| FR-5-6 | ガントチャートの時間軸表示単位を **日 / 週 / 月** から選択できる |

**FR-5-6 補足（表示単位ごとの挙動）：**

| 単位 | 列幅の粒度 | ヘッダ表示例 | 想定プロジェクト規模 |
|------|-----------|------------|-------------------|
| 日 | 1営業日 = 1列 | `4/20（月）` | ～3ヶ月 |
| 週 | 1週 = 1列（月曜起点） | `4/20 週` | 3〜12ヶ月 |
| 月 | 1ヶ月 = 1列 | `2026年4月` | 6ヶ月〜 |

バーの長さは選択単位に応じてスケーリングし、営業日数の比率で描画する。

---

### FR-6: エクスポート

| ID | 要件 |
|----|------|
| FR-6-1 | 現在の設定を埋め込んだスタンドアロン HTML をエクスポートできる |
| FR-6-2 | ガントチャート部分を PNG としてダウンロードできる |
| FR-6-3 | 設定を JSON ファイルとして保存・読み込みできる |

---

### FR-7: 祝日データ管理

| ID | 要件 |
|----|------|
| FR-7-1 | 2024〜2028年分の日本の祝日データを HTML 内にバンドルし、オフラインでも即時利用できる |
| FR-7-2 | ボタン1つで [holidays-jp API](https://holidays-jp.github.io/api/v1/date.json) から最新データを取得・上書きできる |
| FR-7-3 | 会社指定休業日は手動で追加・削除できる |

---

## 非機能要件

| ID | 要件 |
|----|------|
| NFR-1 | バックエンド不要（ブラウザのみで動作する単一 HTML ファイル） |
| NFR-2 | 外部ライブラリは最小限（PNG 出力時のみ html2canvas を CDN から遅延ロード） |
| NFR-3 | 日本語対応（UI・日付表示） |
| NFR-4 | 設定は localStorage に自動保存し、再訪問時に復元する |

## システム設計

### アーキテクチャ概要

```
gantt-generator.html（単一ファイル）
│
├── Sidebar（左ペイン：設定エディタ）
│   ├── Tab: プロジェクト設定  ← projectName, startDate, releaseDate, evalPeriod, ganttUnit
│   ├── Tab: 休日             ← national（自動取得ボタン付き）/ company（手動）
│   ├── Tab: 工程タイプ        ← phaseTypes（名前・チーム・色）
│   ├── Tab: 担当者           ← people（担当工程・参画日・稼働率）
│   └── Tab: アイテム         ← items（フェーズ・工数・担当者割付）
│
├── Preview（右ペイン：プレビュー）
│   ├── 警告バナー
│   ├── ガントチャート（SVG）
│   └── スケジュールサマリー表
│
└── Footer ボタン
    ├── [HTML出力]  [PNG出力]
    └── [JSON保存]  [JSON読込]
```

### データスキーマ（JSON）

```jsonc
{
  "projectName": "string",
  "startDate": "YYYY-MM-DD",
  "releaseDate": "YYYY-MM-DD",

  // FR-1-2: 評価期間（日 / 週 / 月）
  "evalPeriod": { "value": 4, "unit": "weeks" },

  // FR-5-6: 表示単位
  "ganttUnit": "weeks",   // "days" | "weeks" | "months"

  // FR-1-3 / FR-7: 休日
  "holidays": {
    "national": ["YYYY-MM-DD"],  // 国民の祝日（バンドルまたはAPI取得）
    "company":  ["YYYY-MM-DD"]   // 会社指定休業日
  },

  // 工程タイプ定義
  "phaseTypes": [
    { "name": "要件定義", "team": "D&T", "color": "#3B82F6" }
  ],

  // 担当者
  "people": [
    {
      "name": "string",
      "team": "string",
      "phases": ["要件定義", "設計開発"],   // 担当可能フェーズタイプ
      "availableFrom": "YYYY-MM-DD | null",
      "utilization": 1.0,                  // 稼働率 0.0〜1.0
      "note": "string"
    }
  ],

  // 開発アイテム
  "items": [
    {
      "name": "string",
      "category": "string",
      "note": "string",
      "phases": [
        {
          "type": "string",            // phaseTypes.name と対応
          "days": 10,
          "allowedPeople": ["name"],   // 省略時 = 自動割り当て
          "requireAll": false,         // true = allowedPeople 全員同時着手
          "background": false,         // true = 工数消費なし（並行進行）
          "fixedStart": "YYYY-MM-DD"   // 省略時 = 自動配置
        }
      ]
    }
  ],

  // リリース評価フェーズ表示設定
  "evalPhase": { "name": "リリース評価", "color": "#8B5CF6" }
}
```

### スケジューリングアルゴリズム

```
前提:
  - 営業日 = 土日 + national + company 休日を除いた日
  - evalPeriod を営業日数に変換: days→そのまま, weeks→×5, months→暦月分を数える

手順:
  1. 各フェーズに priority = 残パイプライン工数合計（LPT方式）を付与
  2. 各営業日ごとにシミュレーション:
     a. 前日完了タスクを確認、後続タスクをキューへ追加
     b. キューを priority 降順にソート
     c. background タスク: 担当者消費なしで進捗加算
     d. 通常タスク: allowedPeople / requireAll / availableFrom を考慮して担当者を割り当て
  3. 全タスク完了 or releaseDate +余裕 まで繰り返す
  4. 警告収集: evalStart 超過タスク、未スケジュールタスク
```

### SVG ガントチャート列幅設定

| 単位 | 列幅 | ラベル列幅 | ヘッダ行高 |
|------|------|----------|---------|
| days | 28px / 営業日 | 220px | 44px |
| weeks | 70px / 週 | 220px | 44px |
| months | 90px / 月 | 220px | 44px |

### エクスポート設計

| 形式 | 実装方法 |
|------|---------|
| **HTML** | 現在の config を JSON シリアライズして `<script>` に埋め込み、スタンドアロン閲覧 HTML を Blob でダウンロード |
| **PNG** | `html2canvas`（CDN から遅延ロード）でガントチャートカード要素をキャプチャし `canvas.toBlob()` でダウンロード |
| **JSON** | config を `JSON.stringify` して Blob ダウンロード。読込は `FileReader` で parse して反映 |

### 祝日データ戦略

```
起動時: バンドルされた 2024〜2028 年データを即時適用（オフライン対応）

「自動取得」ボタン押下時:
  GET https://holidays-jp.github.io/api/v1/date.json
  → { "YYYY-MM-DD": "祝日名", ... } 形式
  → config.holidays.national を上書き
  → localStorage に保存
```
